Release 0.6.0 is NOT backward compatible with previous releases.
Contact us if you need help migrating from previous releases.
üôå New
  • Fixed Issue #59.
    • Class NCModelDescriptor has been removed and replaced with direct methods on NCModel interface.
    • Model and element builders have been removed and replaced with adapters.
    • Classes NCModelAdapter and NCModelFileAdapter were introduced replacing builder functionality.
    Although this is a significant simplification - this is NOT a backward compatible change and will require changes in your model logic. Look at the examples to learn how to use these new APIs.
  • Fixed Issue #51.
    • New example added - Phone Dialer that is using Google Natural Language tokens.
  • Fixed Issue #53.
  • Fixed Issue #36.
    In this release the processing of built-in tokens has been significantly enhanced and it is NOT backward compatible.
    • Introduced support for new built-in tokens:
      Token Provider Token ID Prefix
      OpenNLP opennlp:
      Stanford CoreNLP stanford:
      Google Natural Language google:
    • NCToken metadata properties NLP_NE and NLP_NNE have been removed.
    • The following NCModel configurations properties, setters and getters, as well as REST error codes have been removed:
      • MAX_GEO_TOKENS
      • MIN_GEO_TOKENS
      • MAX_NUM_TOKENS
      • MIN_NUM_TOKENS
      • MAX_DATE_TOKENS
      • MIN_DATE_TOKENS
      • MAX_FUNCTION_TOKENS
      • MIN_FUNCTION_TOKENS
    • REST /probe/all call return object field model has new array property enabledBuiltInTokens.
    • REST server configuration has new property tokenProviders that provides an array of token providers that should be enabled and whose tokens will be detected by the server. Allowed values are: opennlp, spacy, nlpcraft, stanford and google. Default value if nto specified is nlpcraft. Property can be empty in which case no built-in NE tokens will be detected.
    • Data model has new array property enabledBuiltInTokens that specifies all built-in NE tokens that this model is expecting to be detected. The default value is an empty array in which case no built-in NE tokens will be detected.
ü§ñ Enhancements
  • Fixed Issue #62:
    • System properties or environment variables NLPCRAFT_SERVER_PROBE_LINKS_DOWNLINK and NLPCRAFT_SERVER_PROBE_LINKS_UPLINK can override REST server configuration server.probe.links.downLink and server.probe.links.upLink. It is intended to be used for pre-configuring cloud images.
  • Fixed Issue #61:

    • NLPCraft built-in tokens now have have prefix nlpcraft: instead of nlp: to better align with new named entity tokens from Google, OpenNLP and Stanford CoreNLP:

      Old Name New Name
      nlp:geonlpcraft:geo
      nlp:numnlpcraft:num
      nlp:functionnlpcraft:function
      nlp:coordinatenlpcraft:coordinate
      nlp:datenlpcraft:date
      This change may require an update to existing intents.
  • Fixed Issue #55:
    • The following data model properties have been removed as unused:
      • VENDOR_NAME
      • VENDOR_EMAIL
      • VENDOR_CONTACT
      • VENDOR_URL
      • DOCS_URL
      See Issue #36 for other modifications to the data model properties.
  • Fixed Issue #32, Issue #45:
    • Method NCIntentSolverContext.getTokens() renamed to getVariantTokens().
    • New @@ and !@ intent RULE DSL operations added for collections containment check.
    • New @# and !# intent RULE DSL operations added for map key containment check.
    • New @$ and !$ intent RULE DSL operations added for map value containment check.
    • New intent RULE DSL ~META[key] operand notation for index or key access to the list or map metadata properties.
    • Intent ITEMs have now default min=1, max=1 quantifiers.
    • Intent TERM now has only one ITEM (it was a redundant collection of ITEMs before).
  • Fixed Issue #47:
    • Not-found function in the intent solver now accepts NCContext as a parameter so that the non-found function can have access to the original query.
  • Fixed Issue #72:
    • Data probe configuration file probe.conf is no longer required and default probe configuration can be overridden via system properties.
